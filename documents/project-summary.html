医療系オンライン予約システム — プロジェクトサマリー

医療機関向けに設計・開発したオンライン予約管理システムです。患者向けの予約フローからスタッフ向けの管理ダッシュボードまで、予約・受付・診察履歴の一連の業務をカバーします。個人情報保護法（APPI）への準拠、多言語対応、堅牢なセキュリティ設計を特長とし、本番環境で稼働中です。

3. セキュリティ設計

設計方針

医療情報を扱うシステムとして「デフォルト拒否」を原則に、認証・認可・入力検証・通信保護・監査の各レイヤーで多層防御を実現しています。

認証・認可

  • Firebase Auth ＋ カスタムクレーム（admin）によるロールベースアクセス制御
  • Firestoreセキュリティルールでデフォルト全拒否（allow read, write: if false）を基本とし、コレクション単位で最小権限を付与
  • 予約の作成・キャンセル・照会はCloud Functions経由のみ（クライアントからの直接書き込み不可）
  • 管理者JWT検証をCloud Functions側で実施し、非管理者アクセスを拒否

データ保護

  • Firestoreルールで全フィールドの型・長さをサーバーサイドバリデーション（例: 氏名50文字、症状1,000文字）
  • 診察履歴（visit_histories）は更新・削除不可（Append-Only）で改ざんを防止
  • usersコレクションは本人のみ読み取り可、書き込みは管理者 or 本人の限定フィールドのみ
  • HTML エスケープ処理（escapeHtml / sanitize）による XSS 対策
  • シークレット管理は Firebase Secret Manager で一元化

通信・インフラ

対策内容
CSP（Content Security Policy）default-src 'self' を基本に、許可ドメインを明示的にホワイトリスト。frame-ancestors 'none', object-src 'none'
HSTSmax-age=31536000（1年、サブドメイン含む）
X-Frame-OptionsSAMEORIGIN（クリックジャッキング防止）
Permissions-Policycamera=(), microphone=(), geolocation=()（不要なデバイスAPI無効化）
CORS許可オリジンを本番ドメインに限定
Referrer-Policystrict-origin-when-cross-origin

攻撃対策

  • レート制限：全公開エンドポイントにIPベースの制限（5リクエスト/分/IP）、超過時HTTP 429を返却
  • 二重予約防止：Firestoreトランザクションによるスロット排他制御（TOCTOU攻撃防止）
  • 列挙攻撃防止：verifyReservationは「予約未存在」と「電話番号不一致」で同一エラーを返却し、予約IDの存在推測を遮断
  • 監査ログ：CSVエクスポート等の操作をaudit_logsコレクションに記録（PII非含有）

4. 法規・コンプライアンス対応（個人情報保護法 = APPI）

設計方針

医療系システムとして症状・保険証等の要配慮個人情報を取り扱うため、個人情報保護法の各条文に対応した機能を実装しています。

条文要件実装内容
第20条第2項
要配慮個人情報		 症状等の医療情報取得には明示的同意が必要 予約フォームで hasSensitiveDataConsent フラグを取得。Cloud Functions がサーバー側で同意未取得の予約を拒否。Firestore ルールでも検証
第15条
利用目的の特定		 利用目的を本人に明示 管理者向け利用規約＋プライバシーポリシー同意モーダル。バージョン管理により、ポリシー更新時に再同意を要求
第17条
データ保存期間		 利用目的に必要な範囲での保存 管理画面からデータ保存期間（月単位）と利用目的テキストを設定可能
第28〜34条
本人の権利		 開示・訂正・利用停止の請求権 患者の開示・訂正・利用停止の権利行使機能を実装。問い合わせ先を管理画面から設定可能
特定電子メール法 メール配信には事前同意が必要 リマインダーメールの配信同意チェック（reminderEmailConsent）。Cloud Functions が同意フラグを確認してから送信
改ざん防止 診察記録の真正性確保 診察履歴コレクションは Append-Only（更新・削除不可）。患者情報のスナップショットを自動保存
監査証跡 操作履歴の追跡可能性 監査ログコレクションに操作記録を保存（PII非含有）。管理者のみ閲覧可能

観点ポイント

UX4ステップウィザード・郵便番号自動補完・年齢自動計算・PDF予約票・QRコード付き確認番号など、患者の入力負荷を徹底的に軽減
